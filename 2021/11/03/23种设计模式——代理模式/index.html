代理模式（Proxy）

作用

如字面意思，增加代理对象以对原对象进行进一步控制的策略方案。

结构图

具体实现

抽象类

// 主题类 抽象类 提供客户端所使用的的接口
class Subject {
public:
  using ptr = std::shared_ptr<Subject>;  // 抽象类指针
  virtual void request() = 0;
};

真实实体类

// 真实实体的具体实现
class RealSubject : public Subject {
public:
  using ptr = std::shared_ptr<RealSubject>;  // 真实实体指针

  virtual void request() override { std::cout << "真实实体的具体需求"; }
};

代理类

// 代理类
class Proxy : public Subject {
public:
  using ptr = std::shared_ptr<Proxy>;  // 代理指针

  void request() {
    std::cout << "代理调用：";
    m_subject->request();
  }

private:
  RealSubject::ptr m_subject{std::make_shared<RealSubject>()};  // 代理的真实实体
};

测试案例

// 生成代理对象
auto proxy_ptr{std::make_shared<Proxy>()};

// 调用代理对象的功能
proxy_ptr->request();

总结

代理模式在原有的对象基础上进行了一次封装，对其进行更多的操作如：增加计数、更改权限等等。拥有与原对象相同的接口、对使用方不会造成过多的负担。

应用场景

  1. 远程代理：为一个对象在不同的地址空间提供局部代表，隐藏了一个对象存在不同地址空间的实事。
  2. 虚拟代理：是根据需要创建开销很大的对象。通过它来存放实例化需要很长时间的真实对象。
  3. 安全代理：更改真实对象的访问权限。
  4. 智能指引：调用真实对象时，代理处理另外一些事如智能指针等。

常规操作

git init 初始化

  • 功能描述：创建空仓库或者重新初始化仓库。在已有仓库使用不会覆盖已经存在的东西。
  • 常用参数：-b <分支名称>, 同时添加一条分支并设定名称。
    例：以当前位置初始化一个空仓库。
    git init .
    例：以当前位置初始化一个空仓库并添加一条名为”fname”的分支。
    git init . -b "fname"

git add 暂存

  • 功能描述：将文件添加到缓存区。
  • 常用参数：-A 全部文件，-u 更新缓存区中的文件
    git add <file_1> <file_2>...
    例：将文件添加至缓存区。
    git add file_name.txt

git diff 比较

  • 功能描述：显示文件在缓存区与工作区之间的区别。
  • 常用参数：无。
    git diff <file_1> <file_2>...
    例：查看一个文件缓存区与工作区的区别
    git diff file_name.txt

git commit 提交

  • 功能描述：将缓存区文件提交到本地仓库。
  • 常用参数：-m 提交时添加备注信息。
  • 注：提交前需要使用配置好用户信息。
    例：提交缓存区文件。
    git commit -m "第一次提交"

git reset 回退

  • 功能描述：将版本回退到之前提交过的版本。
  • 常用参数：–hard 清空当前工作区和缓存区的修改，–soft 将当前与旧版本合并
    例：清空工作区和缓存区的修改
    git reset --hard
    例：将当前版本与上一个版本合并
    git reset --soft HEAD^   # ^的数量表示上n个版本

git rm 删除

  • 功能描述：可删除缓存区文件和工作区文件。
  • 常用参数：-f 强制删除，–cached 只删缓存区
    例：将文件从缓存区和工作区删除
    git rm file_name.txt
    例：只删除缓存区中的文件
    git rm --cached file_name.txt

分支操作

git branch 创建分支

  • 功能描述：创建单条分支，不接其他参数时展示所有分支。
  • 常用参数：–list / -a 展示所有分支，-d 删除分支
    例：创建分支 b1。
    git branch b1
    例：查看所有分支。
    git branch
git branch --list
git branch -a
    例：删除b1分支
    git branch b1

git checkout 切换分支

  • 功能描述：切换分支并把工作区内容调整为新分支最后一次提交的内容。
    git checkout new_b

git merge 合并分支

  • 功能描述：将其他分支内容合并到本分支。
  • 注：可能会出现冲突。
    例：将b1分支数据合并到本分支
    git merge

合并冲突

  • 原因：当合并分支内与本分支内容有歧义时，就会在合并时报出合并冲突。
  • 解决方法：当出现冲突时，git会将有歧义的内容都写入在文件内，需要你对文件进行修改。最后使用 git add 提交至缓存区表示告诉git冲突解决了。
